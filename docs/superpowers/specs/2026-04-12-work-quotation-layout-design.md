# 报价模式（Work）前端布局优化 — 设计文档

**日期**：2026-04-12  
**状态**：已批准（用户确认方案 C），待实现  
**范围**：仅控制台前端的 **报价 / Work** 标签页布局与间距；**不改**后端接口与业务逻辑。

---

## 1. 背景与目标

当前 Work 页在 `control-ui/src/ui/views/work.ts` 中主要由 **单列竖排** 内联样式构成：上传区、文字输入区、客户价位与执行按钮依次堆叠。`textarea` 等控件曾使用 **`max-width: 560px`** 等限制，在宽屏下易出现 **内容挤在左侧、右侧大量留白**，且 **区块间距偏小**，整体显得「紧」。

**目标（方案 C）**：

- **A**：宽屏下 **适度加宽** 主内容区并 **水平居中**，减少单侧空白。
- **B**：**统一加大** 垂直与内部间距，提升可读性与层级感。

**成功标准**：

- 宽屏（视口宽度 ≥1280px）下，报价主表单区域 **视觉宽度约 960～1120px 居中**，上传区与文字输入区 **同宽**，无明显「左窄右空」。
- 各主要区块之间间距 **明显大于改版前**（目标约 **20～24px** 量级）；灰底子块内边距 **约 14～16px**（高于当前约 10px）。
- 窄屏与移动端 **不出现横向滚动**、布局不劣于现状；若与 `layout.mobile.css` 冲突则 **做小范围覆盖**。

---

## 2. 非目标（本期不做）

- **双栏布局**（宽屏下「上传 | 文字」并排）：可作为 **后续迭代**，本期不实现。
- 修改 **待选 / 保存报价单** 等流程逻辑；仅在与主卡 **视觉一致** 需要时，对 **同类卡片** 做 **间距 class 复用**。
- 变更 i18n 文案内容（除非为配合布局必须换行，原则上不改）。

---

## 3. 实现策略

**推荐**：在 `components.css`（或项目约定的全局样式入口）中新增 **布局专用 class**（例如 `.work-quotation` / `.work-quotation__inner`），`work.ts` 中为对应 `<section class="card">` 或内层容器 **挂载 class**，**减少** 内联魔法数字。

**备选**：若团队倾向最小 diff，可 **首版仅改 `work.ts` 内联样式**；仍建议后续抽 class 与全局样式对齐。

---

## 4. 布局与尺寸规范

### 4.1 内容宽度

- 在 **报价主卡片**内增加 **内层容器**（或等价结构）：
  - `max-width: 1120px`（或 `min(1120px, 100%)`）；
  - `margin-left: auto; margin-right: auto`；
  - 上传区、文字区、价位与执行控制区 **均置于该容器内**，**全宽** 填充容器。
- **移除或放宽** 文字区 `textarea` 上 **单独限制宽度** 的 `max-width: 560px`，使其与容器同宽（保留 `width: 100%`）。

### 4.2 间距（垂直节奏）

- **卡片内**：标题 / 副标题与第一个功能区块之间、**主要功能区块之间**：目标 **20～24px**（替换当前多处约 **12px** 的 `margin-bottom` 等，以设计实现为准）。
- **上传区、文字区** 等灰底子块：内边距 **14～16px**（当前约 10px）；圆角可与现有 **8px** 保持一致或略统一为设计 token。
- **文字区**：可适当增加 **`rows`**（例如 **5～6**）或设置 **`min-height`**，避免输入区过扁（具体数值以实现时视觉为准）。

### 4.3 执行行

- 「执行 / 取消 / 重试」与 **状态文案** 一行：与上方控件区块 **增加间距**；宽屏若 **状态文字过长**，允许 **换行** 或 **与按钮组折行**，避免挤压主按钮（实现时二选一，以不溢出为准）。
