| 2寸 | DN50 / 2" | 2英寸管径 |
| 1寸 | DN25 / 1" | 1英寸管径 |
| 6英寸 | DN150 / 6" | 6英寸管径 |
| 1½寸 | DN40 / 1.5" | 1.5英寸（英制1-1/2"） |
| 3/4寸 | DN20 / 3/4" | 3/4英寸 |

### 英制规格（非标准写法）
| 询价词 | 正确对应 | 说明 |
|--------|----------|------|
| 1½ in | DN40 / 1-1/2" | 带空格/无引号 |
| 3/4" | DN20 / 3/4" | 标准写法（但未统一处理） |
| 1.5"" | DN40 / 1-1/2" | 多余引号 |

## 根因分析

### 问题1: 中文尺寸"寸"未映射

在 `wanding_fuzzy_matcher.py` 的 `_expand_unit_tokens` 函数中：

```python
# 当前实现的映射
MM_TO_INCH = {
    "16": '1/2"', "20": '3/4"', "25": '1"', "32": '1-1/4"',
    "40": '1-1/2"', "50": '2"', "65": '2-1/2"', ...
}
INCH_TO_MM = {v: k for k, v in MM_TO_INCH.items()}

# 问题：缺少中文"寸"到 DN 的映射
# "2寸" → DN50 → 2"  没有被识别
# "1寸" → DN25 → 1"  没有被识别
```

### 问题2: 英制规格处理不完整

`_split_tokens` 函数可以识别 `1-1/2"`、`3/4"` 等，但以下格式有问题：
- `1½ in` → 空格分隔，应合并为 `1-1/2"` 再处理
- `1.5""` → 多余引号
- `1½寸` → 中文"寸"混合英制分数

## 优化方案

### 方案: 在 `_expand_unit_tokens` 中增加中文尺寸映射

需要在 `wanding_fuzzy_matcher.py` 中添加：

```python
# 中文尺寸制式 → DN(mm)
CUN_TO_MM = {
    "寸": "25",      # 1寸 ≈ DN25
    "2寸": "50",     # 2寸 ≈ DN50
    "6寸": "150",    # 6寸 ≈ DN150
    "1½寸": "40",    # 1.5寸 ≈ DN40
    "3/4寸": "20",   # 3/4寸 ≈ DN20
}

# 同时需要在 _split_tokens 中识别中文尺寸
# 在识别纯数字之后，识别类似 "2寸"、"6英寸" 这样的模式
```

### 实施步骤

1. 在 `wanding_fuzzy_matcher.py` 添加 `CUN_TO_MM` 映射
2. 修改 `_split_tokens` 函数，识别 "X寸"、"X英寸" 模式
3. 修改 `_expand_unit_tokens` 函数，展开中文尺寸为 DN 和 inch
4. 在 `wanding_business_knowledge.md` 中补充字段匹配规则
5. 验证不影响其他查询的正确率
