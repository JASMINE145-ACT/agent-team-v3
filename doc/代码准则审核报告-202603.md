## 二、按优先级分类

### P0（阻塞/一致性，建议优先修）

- **依赖与静默异常**  
  - core 依赖 Config（agent.py, llm_client.py）→ 改为参数注入。  
  - agent.py L234-235 无声吞异常 → 至少 logger.warning(..., exc_info=True)。  
  - remember.py 三处 except pass → 至少 logger.warning 或 exception。  
  - quotation_agent_tool.py 直接 DataService() → 改为 get_oos_data_service() 或注入。  
- **API 返回格式**  
  - routes_health：/health 与 price-levels 失败 → 统一为 success/data/error；失败用 error 不用 data:[]。  
  - routes_upload：两处成功返回 → 包成 success + data。

### P1（可读与可维护）

- **单例与注释**：agent/tools.py 从 deps 取 DataService → import 提到顶部并加「例外」注释；deps 单例可考虑 lock、补返回类型。  
- **重复逻辑**：quotation handler、quote_tools、quotation_agent_tool 中重复 unwrap/success 逻辑 → 统一用 unwrap_tool_result 或单一 helper。  
- **异常与类型**：llm_client、context_compression、quote_tools、inventory_agent_tools、data_service 等处 → 该用 exc_info 的加上、该打 warning 的升级；公共函数补类型注解。  
- **Server 格式**：routes_oos 的 deleted/message、routes_procurement 的 approved_count/message → 纳入 data。  
- **前端**：work.ts 上传移到 Controller；移除 PRICE_LEVEL_OPTIONS，仅用 API 档位。

### P2（优化与结构）

- **单文件超 200 行**：agent.py、tools.py、data_service.py、quote_tools.py、inventory_agent_tools.py、agent_runner.py → 按领域或职责拆分。  
- **全局变量**：ALL_TOOLS、inventory 的 _table_agent/_resolver 等 → 优先 app.state 或注入，短期加注释。  
- **类型与 import**：registry/extension 的 Callable 完整注解；函数内非延迟/非破循环的 import 移到顶部。

## 五、P2 实施记录（已落实）

- **Callable 类型**：`backend/core/registry.py` 增加 `ToolHandler = Callable[..., Awaitable[str]]`，`ToolEntry.handler` 与 `register(..., handler)` 使用该类型；`extension.py` 的 `register_tool(..., handler: ToolHandler)` 同步。
- **import 上移**：`routes_health.py` 将 price-levels 用到的 `PRICE_LEVEL_DISPLAY_NAMES`/`PRICE_COLS` 移至顶部；`routes_upload.py` 将 `text_to_inquiry_items`/`fill_template_with_inquiry_items` 移至顶部；`routes_oos.py` 将 `inv_config`、`_BUSINESS_KNOWLEDGE`、`invalidate_business_knowledge_cache`、`persist_out_of_stock_records` 移至顶部；并修正 `shortage_delete` 返回为 `data: { deleted }`。
- **全局与单例**：`agent/tools.py` 为 `ALL_TOOLS` 增加注释「延迟单例，见代码准则规则3；多 worker 或需重置时建议迁至 app.state 或由 Extension 注入」；`deps.py` 单例增加 `threading.Lock()` 双检锁，多 worker 安全，`reset_for_testing` 在 lock 内重置。
- **agent.py 拆分**：新增 `backend/core/agent_helpers.py`，迁出 `_extract_tag`、`build_system_prompt`、`call_llm_streaming_sync`；agent 复用 `context_compression._trim_context`，删除本文件内重复实现；`agent.py` 行数降至约 260 行以内。
- **tools.py 拆分**：新增 `backend/agent/tools_oos.py`，迁出无货相关 `_run_oos_list`、`_run_oos_stats`、`_run_oos_by_file`、`_run_oos_by_time`、`_run_register_oos`；`tools.py` 从该模块导入并保留 EXTRA_TOOLS schema，行数降至约 180 行以内。
- **未做**：`data_service.py`（~1070 行）、`quote_tools.py`（~556 行）、`agent_runner.py`（~303 行）、`inventory_agent_tools.py`（~336 行）的进一步按领域拆分，以及 inventory 模块级 `_table_agent`/`_resolver` 收口至 app.state，留作后续迭代。

## 六、Neon 业务 Skill 与 routes 分层（补充）

| Skill | 依赖后端模块 / 路由 | Neon 表 |
