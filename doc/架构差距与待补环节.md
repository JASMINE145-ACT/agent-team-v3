# Agent Team version3 — 架构差距与待补环节

> 对照《项目方向与目标_基于Coze企划》与现有实现，列出**还差的环节与功能**。  
> 说明：`项目架构3.0(1).docx` 为二进制无法直接解析，本差距分析以 `项目方向与目标_基于Coze企划.md` 及当前代码为准。

---

## 一、已具备的环节（现状）

| 环节 | 已有能力 |
|------|----------|
| **报价** | 万鼎选型、历史匹配、Work 填表；报价落库（quotation-drafts）、from-text；待确认报价单带编号 |
| **成单** | 成单页勾选报价单 → PATCH confirm → `run_fulfill_loop` → **本地** orders/order_lines，返回 order_no（**未写 ACCURATE，未真实锁库存**） |
| **无货/缺货** | 登记、看板、邮件；ShortageRecordDB；缺货列表 GET /api/shortage/list |
| **采购批准** | 前端勾选缺货项 → POST /api/procurement/approve → 落库 + 发邮件给采购员 |
| **补货** | 补货单草稿、预览、PATCH confirm（可调 modify_inventory supplement 写 ACCURATE） |
| **库存查询** | inventory 查询（按 code/关键词）、无货/缺货登记 |
| **控制台** | 报价/成单/采购三页、聊天、上传、Work 运行与 resume |

---

## 二、还差的环节 / 功能

### 2.1 成单闭环（与架构目标差距）

| 差距 | 说明 |
|------|------|
| **订单写入 ACCURATE** | 当前仅写本地 `orders` / `order_lines`，未调用 ACCURATE 订单 API。需对接「成单 → 写 ACCURATE 订单」并返回 ACCURATE 订单号。 |
| **真实库存锁定** | 成单后未在 ACCURATE 或本地执行「锁库存/占位」，存在超卖风险。需在 fulfill 或下游增加锁库调用。 |

### 2.2 库存与缺货（与架构目标差距）

| 差距 | 说明 |
|------|------|
| **缺货时推荐替代型号** | 企划要求：缺货时能推荐同规格/可兼容型号并带库存与报价。当前无此能力。 |
| **查询记录落库与分析** | 库存查询记录未系统落库，无法做「查询频次分析、库存布局优化」。 |

### 2.3 采购闭环（与架构目标差距）

| 差距 | 说明 |
|------|------|
| **采购需求/采购单与 ACCURATE** | 当前仅有「缺货批准 → 落库 + 邮件」，无采购单草稿、无写入 ACCURATE 采购模块。企划要求：无货/缺货或成单锁库不足时生成采购需求或采购单并写入 ACCURATE（若已对接）。 |
| **采购进度与入库** | 无「待下单/已下单/待到货」等进度录入与查询；到货后未更新库存与订单状态（采购-入库闭环）。 |

### 2.4 送货/物流闭环（当前缺失）

| 差距 | 说明 |
|------|------|
| **送货单与物流** | 无送货单生成、物流单号绑定、物流状态录入/查询。 |
| **收货确认与订单回写** | 无「客户确认收货」及回写订单状态为「已收货」并同步 ACCURATE。 |

### 2.5 收款闭环（当前缺失）

| 差距 | 说明 |
|------|------|
| **收款提醒** | 无按订单付款条款（如预付款 30%）的定时提醒。 |
| **到账确认与状态** | 无「已付款/已到账 + 订单号」的确认与落库，无订单状态更新为「已收款」。 |
| **与 ACCURATE 财务同步** | 收款数据未与 ACCURATE 财务模块同步。 |

### 2.6 企微对接（当前缺失）

| 差距 | 说明 |
|------|------|
| **企微作为主入口** | 当前以控制台/聊天为主；无企微 API（消息接收、回复、群@、机器人/应用）对接。 |
| **话术与指令对齐** | 企划中的「报价+型号+数量」「库存查询+型号」等指令格式未在企微侧落地。 |

### 2.7 CEO 审批模式（设计已有，未实现）

| 差距 | 说明 |
|------|------|
| **后端** | 无 `POST /api/ceo/decide`，无 `ceo_decision` 服务。 |
| **前端** | 无「人工 / CEO」模式开关，确认/批准前未先调 CEO 再调原 API。 |

### 2.8 数据与运维（与架构目标差距）

| 差距 | 说明 |
|------|------|
| **与 ACCURATE 同步规则** | 库存/订单/采购/收款的同步节奏、失败重试与告警未成文或未实现。 |
| **合规与审计** | 数据同步失败告警、敏感数据不落日志等需按企划补全。 |

## 三、按优先级的待补清单（与企划 P0/P1/P2 对齐）

| 优先级 | 待补项 |
|--------|--------|
| **P0** | ① 成单写 ACCURATE + 真实锁库存；② 缺货替代型号推荐；③ 无货/缺货与采购需求或采购单衔接（至少生成采购需求/草稿并可写 ACCURATE）；④（可选）CEO 审批模式落地。 |
| **P1** | ① 收款提醒与到账确认、收款落库并同步 ACCURATE；② 企微入口对接（若业务需要）。 |
| **P2** | ① 采购单全流程（生成→跟进→入库→更新库存与订单）；② 送货单与物流状态、收货确认与订单回写；③ 数据同步监控、异常告警与合规审计。 |

---

## 四、小结

- **已有**：报价→报价落库→成单（本地订单）、缺货列表→采购批准、补货确认、库存查询、无货/缺货登记与邮件、控制台三页与 Work。
- **还差**：成单与 ACCURATE/锁库存、缺货替代推荐、采购单/进度/入库、送货与物流、收款全环节、企微入口、CEO 审批实现、以及数据同步与合规层面的完善。

若 `项目架构3.0(1).docx` 中有额外模块或流程（如具体流程图、接口表），可将其中的「目标状态」补充进本文档，再对上述清单做增删即可。
