## 三、优化待办清单

### 🔴 P0 - 必须修复

#### 1. 波纹管匹配逻辑优化
**问题**：波纹管（corrugated pipe）和 PPR 管、PVC 电工套管是**完全不同的产品线**，但系统把它们混在一起匹配，导致 Row 16/37/43/44 四次查询全部失败。

**现状**：
- 搜索"双壁波纹管10KN DN300mm" → 匹配到普通 PVC 给水管
- 搜索"波纹管 DN20" → 匹配到 PPR 冷水管 / 电工套管

**建议方案**：
```
在 match_quotation 中增加产品类型预判：
如果 keywords 包含"波纹管"，强制优先匹配 8020035095 系列 (SN4/SN8 双壁波纹管)

产品类型词表：
- 波纹管 / corrugated → 8020035095/8020035096/8020035097 系列
- PVC管 / PPR管 → 801001/801006 系列
- 电工套管 → 803002 系列
```

**验证标准**：用"双壁波纹管10KN DN300mm"查询，返回结果的产品名称必须包含"双壁波纹管"或"corrugated"关键词。

---

#### 2. 缺少 price 参数时的追问机制
**问题**：Row 50 "检查利润" 没有 price 参数就结束了，用户不知道要提供价格。

**建议方案**：当用户查询利润/利润率，但没有提供成交价时，自动追问：
```
用户查询利润/利润率，但没有提供成交价，请问您的成交价/报单价是多少？
```

---

#### 3. OCR 截断处理
**问题**：Row 45 的 extra 中 OCR 结果被截断，只显示了 3 个产品，缺少第 4 个的信息。

**建议方案**：
- 检查 OCR 解析是否有截断限制
- 在 extra 中标记"共 N 个产品，已列出前 M 个"

### 🟡 P1 - 重要优化

#### 4. 匹配失败时的参数提示
**问题**：当没有自动锁定唯一匹配时（如 Row 39/43/44），用户不知道该**输入什么**来区分。

**建议方案**：当 pending > 0 时，自动给出**下一个应该明确的参数**提示：
```
提示示例："双壁波纹管 DN300mm" 如有多条候选，提示"请明确压力等级(SN4/SN8)或长度"
```

---

#### 5. 重复查询优化
**问题**：Row 18/20/28/30/32/35/68 共 **7 次完全相同**的查询"直接50 价格"，说明系统没有记住上下文或用户没有得到满意答案。

**建议方案**：
- 如果上一轮已查询过且用户没有新要求，直接复用结果而非重新查询
- 在 thinking 中标注"复用上一轮查询结果"

---

#### 6. PR vs PPR 拼写纠正
**问题**：Row 76 Query 写的是"PR 冷水管"（少一个 P），系统没有纠正就直接按 PPR 处理。

**建议方案**：
- 对常见拼写错误做预处理纠正（PR → PPR）
- 或在 thinking 中标注"已自动纠正 PR → PPR"
